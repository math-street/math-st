# A019 - Build an exact order-r class from \(1-4\cdot2^r\)

## Idea

- [CONJECTURE] A self-contained imaginary-quadratic construction with a
  visibly exact order-\(r\) ideal class might provide the missing target of
  A001 before the evaluator is considered.

## Construction

- [PROVED] Let \(r\ge3\) be prime, put
  \(\Delta_r=1-4\cdot2^r<0\), and let
  $$
  \mathcal O_r=\mathbb Z[\alpha],\qquad
  \alpha=\frac{1+\sqrt{\Delta_r}}2,
  \qquad \alpha^2-\alpha+2^r=0.
  $$
  Since \(\Delta_r\equiv1\pmod4\), this is an imaginary-quadratic order of
  discriminant \(\Delta_r\).
- [PROVED] The ideal \(\mathfrak a=(2,\alpha)\) has norm \(2\), because
  \(\mathcal O_r/\mathfrak a\simeq\mathbb F_2\).
- [PROVED] The ideal is invertible.  Its conjugate is
  \(\overline{\mathfrak a}=(2,1-\alpha)\), and their product is generated by
  \(4,2\alpha,2(1-\alpha),\alpha(1-\alpha)=2^r\).  Every generator lies in
  \((2)\), while \(2\alpha+2(1-\alpha)=2\), so
  \(\mathfrak a\overline{\mathfrak a}=(2)\) and
  \(\mathfrak a^{-1}=\tfrac12\overline{\mathfrak a}\).

## Exact order proof

- [PROVED] For \(1\le k\le r\),
  $$
  \mathfrak a^k=(2^k,\alpha).
  $$
  The case \(k=1\) is the definition.  If the identity holds at \(k<r\),
  multiplying by \((2,\alpha)\) gives generators
  \(2^{k+1},2^k\alpha,2\alpha,\alpha^2\).  The relation
  \(\alpha^2=\alpha-2^r\), with \(2^{k+1}\mid2^r\), puts \(\alpha\) in the
  product and proves equality; both inclusions then follow from the displayed
  generators.
- [PROVED] At \(k=r\),
  $$
  \mathfrak a^r=(2^r,\alpha)=(\alpha),
  $$
  because \(\alpha\mathcal O_r=(\alpha,\alpha^2)) and
  \(\alpha-\alpha^2=2^r\).  Thus the class of \(\mathfrak a\) has order
  dividing \(r\).
- [PROVED] The ideal \(\mathfrak a\) is not principal.  If
  \(\mathfrak a=(\beta)\), then \(|N(\beta)|=2\).  Write
  \(\beta=u+v\alpha\) with \(u,v\in\mathbb Z\).  Then
  $$
  N(\beta)=\frac{(2u+v)^2+|\Delta_r|v^2}{4}.
  $$
  If \(v\ne0\), this is at least \(|\Delta_r|/4>2\); if \(v=0\), it is the
  integer square \(u^2\ne2\).  Both alternatives are impossible.
- [PROVED] Since \(r\) is prime, \([\mathfrak a]\ne1\), and
  \([\mathfrak a]^r=1\), the class \([\mathfrak a]\) has exact order \(r\).
- [PROVED] The corresponding primitive positive binary quadratic form is
  \([2,1,2^{r-1}]\), whose discriminant is \(\Delta_r\).

## Size audit

- [PROVED] The discriminant bit length satisfies
  $$
  r+1<\log_2|\Delta_r|<r+2.
  $$
  With \(n=\lceil\log_2r\rceil\), this is
  \(2^{\Theta(n)}\), not \(n^{O(1)}\).
- [PROVED] The construction violates SG-01's full input-length requirement
  and lies far outside the SG-25 Hafner--McCurley target window
  \(B=o(n^2/\log n)\).

## Outcome

- [PROVED] Exact order-\(r\) class construction is not intrinsically the
  problem: the displayed family supplies one uniformly, but only in an
  exponentially long target order.
- [PROVED] A viable target construction must instead output a discriminant of
  polynomial bit length, within SG-25's narrower window for the checked target
  algorithm, together with a certificate that a supplied class has exact
  order \(r\).

## Post-mortem

**Why it failed:** [PROVED] Encoding \(2^r\) requires \(\Theta(r)\) bits,
which is exponential in the source security parameter \(\log r\).

**What transfers:** [PROVED] The ideal-power and nonprincipality proofs give a
clean exact-order test fixture and separate target construction from evaluator
construction.

**Would it work under different assumptions?** [PROVED] It works if setup of
size exponential in \(\log r\) is allowed, but that assumption explicitly
violates the operational transfer definition.
